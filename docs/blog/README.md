# Blog Structure

This folder contains blog posts published according to the [amplification plan](../amplification-plan.md).

## Naming Convention

Use descriptive filenames:

- `YYYY-MM-DD-slug.md` (e.g., `2025-01-15-baseline-is-the-goal.md`)
- Or just `slug.md` if you prefer (e.g., `baseline-is-the-goal.md`)

## Format

Each post should:

- Be self-contained (no required reading order)
- Include a clear title and date
- Capture explicit learning or clarify doctrine
- Link back to relevant docs (`../doctrine.md`, `../operating-manual.md`, etc.)

## Index

Add new posts to [`index.md`](./index.md) with a brief description.

## Sharing

GitHub is the canonical source. Link directly to posts:

- **Individual post**: `https://github.com/yourusername/becoming-engine-v/blob/main/docs/blog/YYYY-MM-DD-slug.md`
- **Blog index**: `https://github.com/yourusername/becoming-engine-v/blob/main/docs/blog/index.md`

**Workflow:**

1. Write post in `docs/blog/`
2. Update `index.md` with the new post
3. Optionally cross-post to broadcast channel (X/Bluesky) linking back to GitHub
